FP review

  • Pure function:
    • It has no side causes: its return value depends strictly on its parameters
    • Its has no side effects: it does not modify any data other than its local variables
    • It is referentially transparent: its call could be replaced by its return value
  • Advantages of a pure function:
    • Fully predictable & reliable: will always return the same for the same parameters
    • Simple to reason about: does not depend on the external world
    • Simple to test: requires no setup, just invocation
    • Avoids all bugs that are caused by side-effects
    • Can be optimized, paralellized, composed, chained...

Let's program with Pure Functions

  • We are now convinced that FP is great
  • Side effects are bad
  • I/O is a side effect
  • ...But applications without I/O are not very useful

Solution

  • Use a functional-oriented framework / runtime
  • Only the framework can do I/O
  • Application code remains 100% pure functional

Elm architecture in JS

  • Redux is a JavaScript implementation of the Elm architecture
    • Reduction in FP: take two or more values and create a new single value out of them
    • Thus, the update function is called the reducer: it creates a new model from an action and previous model
  • Redux is framework-neutral, but used mainly in combination with React
  • Other implementations of this pattern:
    • Vuex for Vue.js
    • ng-redux / ng2-redux for Angular
  • Time-travel debugging
    • Vuex for Vue.js
    • ng-redux / ng2-redux for Angular
